# Implementation Plan: Snake Web Game

**Branch**: `001-snake-web-game` | **Date**: 2026-04-11 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/001-snake-web-game/spec.md`

## Summary

Build a complete browser-based Snake game using Phaser 4.0.0 (bundled locally as `lib/phaser.min.js`),
HTML5, and CSS3. Three Phaser Scenes (MenuScene, GameScene, GameOverScene) encapsulate the start
screen, core game loop, and end screen respectively. All game code lives in a single `game.js` file
to avoid ES-module CORS restrictions when opened via `file://`. No build step or internet connection
required at runtime. Target audience: children aged 8–14.

## Technical Context

**Language/Version**: Vanilla JavaScript ES6+ (no transpilation)
**Primary Dependencies**: Phaser 4.0.0 (local copy at `lib/phaser.min.js`)
**Storage**: N/A — personal best is passed via Phaser Scene `data` object (session-only, resets on page reload)
**Testing**: Manual browser testing (open `index.html`, play through scenarios)
**Target Platform**: Modern evergreen browsers — Chrome 90+, Firefox 88+, Edge 90+, Safari 14+; desktop and tablet (≥ 768 px viewport)
**Project Type**: Single-page browser game (static files, no backend)
**Performance Goals**: Smooth gameplay at configured tick rate (default 150 ms), no perceptible stutter on evergreen browsers
**Constraints**: Must work via `file://` protocol; no npm, no build tooling, no CDN calls at runtime; Phaser bundled locally
**Scale/Scope**: Single HTML page + CSS + JS + one local library file (~4 files total)

## Phaser Architecture

### Scenes

| Scene Key | File | Responsibility |
|-----------|------|----------------|
| `MenuScene` | `game.js` | Start screen — title, instructions, Play button, personal best display |
| `GameScene` | `game.js` | Core game loop — snake movement, collision detection, food spawn, HUD, speed scaling |
| `GameOverScene` | `game.js` | End screen — final score, personal best, new-record celebration, Play Again button |

### Shared Constants (top of `game.js`)

```
CELL = 28          // grid cell size in pixels
COLS = 20          // grid columns
ROWS = 20          // grid rows
CANVAS_W = 560     // COLS × CELL (560 px)
CANVAS_H = 600     // ROWS × CELL + HUD_H (560 + 40 = 600)
HUD_H = 40         // height reserved above grid for score bar
TICK_BASE = 150    // initial tick delay in ms
TICK_MIN = 80      // minimum tick delay (max speed) in ms
TICK_STEP = 5      // ms reduction per speed-up event
SCORE_PER_SPEEDUP = 5  // points between speed-up events
```

### Color Palette

### Input Mapping

| Input | Direction |
|-------|-----------|
| `ArrowUp` / `W` | UP `{x:0, y:-1}` |
| `ArrowDown` / `S` | DOWN `{x:0, y:+1}` |
| `ArrowLeft` / `A` | LEFT `{x:-1, y:0}` |
| `ArrowRight` / `D` | RIGHT `{x:+1, y:0}` |
| `Space` / `Enter` | Start / Restart (MenuScene + GameOverScene) |

Pressing the direction directly opposite to `state.direction` is silently ignored.

### Game-Loop Tick Logic (GameScene)

```
tick():
  1. Apply nextDirection → direction
  2. newHead = { x: head.x + dir.x, y: head.y + dir.y }
  3. Wall collision?  newHead.x < 0 OR >= COLS OR newHead.y < 0 OR >= ROWS  → gameOver()
  4. Self collision?  snake.some(s => s.x===newHead.x && s.y===newHead.y)   → gameOver()
  5. Food collision?  newHead matches food pos → score++, growing=true, spawnFood(), updateSpeed(), updateHUD()
  6. If NOT growing: snake.pop()   (remove tail segment)
  7. snake.unshift(newHead)         (prepend new head)
  8. Win?  snake.length === COLS*ROWS  → gameWon()
  9. redraw()
```

### CSS Styling Rules

- `body`: `background: linear-gradient(135deg, #1a1a2e 0%, #16213e 50%, #0f3460 100%)` — full viewport, flex centering
- `canvas`: `border-radius: 12px`, `box-shadow: 0 0 40px rgba(0,230,118,0.3)` — green glow matching snake head color
- No horizontal overflow at any viewport ≥ 768 px
- Responsive: canvas uses `max-width: 100%; height: auto` via CSS when viewport is narrower than canvas width

### Button Helper (`makeButton`)

```
makeButton(scene, cx, cy, label, bgColor, textColor='#ffffff', width=200, height=64)
  → Draw rounded-rect background with Graphics (fillStyle bgColor, fillRoundedRect)
  → Add centered text (bold, 26px, textColor)
  → Set interactive, pointer cursor
  → Add tween on pointerover  (scale 1.08, brightness)
  → Add tween on pointerout   (scale 1.0)
  → Add tween on pointerdown  (scale 0.95)
  → Return { bg: Graphics, label: Text }
```

4. Write top-of-file constants block in `game.js`: CELL, COLS, ROWS, CANVAS_W, CANVAS_H, HUD_H, tick timing, color palette
5. Write DIRS + OPPOSITES objects
6. Write `makeButton()` helper function

### Phase 3 — MenuScene (depends on Phase 2; blocks nothing)

7. `class MenuScene extends Phaser.Scene`
8. `create()`: HUD bar, title with pulse tween, instruction text, Play button, personal best display
9. Keyboard shortcuts: Space + Enter → start GameScene

### Phase 4 — GameScene (depends on Phase 2; core feature)

10. `class GameScene extends Phaser.Scene`; `create(data)`: state init, Graphics, HUD texts, input setup, food spawn, tick start
11. `update()`: buffer input into `nextDirection`
12. `tick()`: full game-loop step as defined above
13. `redraw()`: clear + draw HUD bar + grid cells + food + snake body + snake head
14. `spawnFood()`: filter empty cells, pick random
15. `updateSpeed()`: reduce tick delay every SCORE_PER_SPEEDUP points; remove + recreate timer
16. `updateHUD()`: refresh score and personal-best texts
17. `gameOver()`: stop tick, red flash overlay tween, transition to GameOverScene after 800 ms
18. `gameWon()`: stop tick, rainbow flash, transition to GameOverScene with win flag

### Phase 5 — GameOverScene (depends on Phase 2; parallel with Phase 3)

19. `class GameOverScene extends Phaser.Scene`; `create(data)`
20. Draw darkened overlay panel
21. Title text (FIM DE JOGO! or VOCÊ VENCEU! 🏆), score, personal best, new-record celebration
22. Play Again button + Space/Enter shortcut

### Phase 6 — Bootstrap (depends on all Scene classes)

23. Define `const config` and call `new Phaser.Game(config)` — last lines of `game.js`
